# TanStack Router Setup Skill

TanStack Router version: 1.x

This skill defines the standard procedure for setting up TanStack Router in a Vite-based React project.

## Prerequisites
- Vite-based project (React/TypeScript)
- `@vitejs/plugin-react` installed

## Installation

```bash
bun add @tanstack/react-router
bun add -D @tanstack/router-plugin
bun add -D @tanstack/react-router-devtools
```

## Vite Plugin Configuration

In `vite.config.ts`, add the router plugin **before** the React plugin:

```typescript
import { tanstackRouter } from '@tanstack/router-plugin/vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [
    tanstackRouter({
      target: 'react',
      autoCodeSplitting: true,
    }),
    react(),
  ],
})
```

> `autoCodeSplitting: true` enables automatic route-level lazy loading without manual `React.lazy()` calls.

## Routes Directory Structure

```
src/routes/
  __root.tsx           # Root route (layout + global providers)
  index.tsx            # Index route (path: /)
  login/
    index.tsx          # /login/
  dashboard/
    index.tsx          # /dashboard/
    $workspaceId.tsx   # /dashboard/$workspaceId (dynamic param)
  settings/
    index.tsx          # /settings/
```

## Root Route (`src/routes/__root.tsx`)

```typescript
import { createRootRoute, Outlet } from '@tanstack/react-router'

export const Route = createRootRoute({
  component: () => (
    <>
      <Outlet />
      <TanStackRouterDevtools />
    </>
  ),
})
```

## Route Files

Each route uses `createFileRoute` with the path matching its file location:

```typescript
// src/routes/index.tsx
import { createFileRoute } from '@tanstack/react-router'

export const Route = createFileRoute('/')({
  component: IndexPage,
})

function IndexPage() {
  return <div>Home</div>
}
```

```typescript
// src/routes/login/index.tsx
import { createFileRoute } from '@tanstack/react-router'

export const Route = createFileRoute('/login/')({
  component: LoginPage,
})
```

## Route with Path Params

Use `$` prefix in filename (e.g., `$workspaceId.tsx`). Access params via `Route.useParams()`:

```typescript
// src/routes/dashboard/$workspaceId.tsx
import { createFileRoute } from '@tanstack/react-router'

export const Route = createFileRoute('/dashboard/$workspaceId')({
  component: WorkspaceDetailPage,
})

function WorkspaceDetailPage() {
  const { workspaceId } = Route.useParams()
  return <div>Workspace: {workspaceId}</div>
}
```

## Router Setup & Provider (`src/main.tsx`)

```typescript
import { RouterProvider, createRouter } from '@tanstack/react-router'
import { routeTree } from './routeTree.gen'

const router = createRouter({ routeTree })

declare module '@tanstack/react-router' {
  interface Register {
    router: typeof router
  }
}

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <RouterProvider router={router} />
  </StrictMode>,
)
```

> The `routeTree.gen.ts` file is auto-generated by the Vite plugin. **Do not edit manually.**

## Navigation

### Declarative (`Link`)

```tsx
import { Link } from '@tanstack/react-router'

// Basic link
<Link to="/dashboard">Dashboard</Link>

// Link with path param
<Link to="/dashboard/$workspaceId" params={{ workspaceId: workspace.id }}>
  {workspace.name}
</Link>
```

### Imperative (`useNavigate`)

```typescript
import { useNavigate } from '@tanstack/react-router'

const navigate = useNavigate()

// Navigate to path
navigate({ to: '/dashboard' })

// Navigate with params
navigate({ to: '/dashboard/$workspaceId', params: { workspaceId: id } })
```

## Auth Guards (Component-Level)

Use a wrapper component in the root route instead of route-level guards:

```typescript
// src/routes/__root.tsx
function AuthGate({ children }: { children: React.ReactNode }) {
  const { isAuthenticated } = useAuth()
  const navigate = useNavigate()
  const location = useLocation()

  useEffect(() => {
    if (!isAuthenticated && location.pathname !== '/login') {
      navigate({ to: '/login' })
    }
  }, [isAuthenticated, location.pathname, navigate])

  if (!isAuthenticated && location.pathname !== '/login') {
    return null
  }

  return <>{children}</>
}

export const Route = createRootRoute({
  component: () => (
    <AuthProvider>
      <AuthGate>
        <Outlet />
      </AuthGate>
      <TanStackRouterDevtools />
    </AuthProvider>
  ),
})
```

## Data Fetching

Do **not** use route loaders. Fetch data inside route components using TanStack Query hooks:

```typescript
function WorkspaceDetailPage() {
  const { workspaceId } = Route.useParams()
  const { data, isLoading } = useEntity(workspaceId)

  if (isLoading) return <Skeleton />
  return <div>{data.name}</div>
}
```

## Validation
- Run `bun tsc` to verify route types.
- Run `bun run build` to verify route tree generation and code splitting.
- Test navigation between all routes (params, links, imperative).
- Test auth guard redirects for protected routes.
